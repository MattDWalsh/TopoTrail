TopoTrail

What's TopoTrail's future?

  • A break from this code!
    I'm a sleepy boi.
  • Satellite imagery as a texture, with the trail overlaid on that
    Affordable sourcing of this data in a simple format doesn't exist at necessary resolutions. USGS has this data available, but due to delivery format and necessary manipulation the timeframe did not allow for its use.
  • Higher resolution meshes
    This is already built, but is too inefficient to expose to end users.
  • Potentially switching mesh format
    OBJ is simple to write, but GLTF may have advantages - including efficiency advantages that would make higher resolution meshes viable.
  • Mesh "normal" smoothing
    "Normals" tell a 3D renderer how to render reflections. The generation process is more complicated than it initially seems.
  • More robust information about each trail
    Some things like trail length, elevation gain, etc. can be programatically generated. Other information may be best sourced through API integration with existing resources.
  • Espansion of user system
    Features like email confirmation upon registration, captchas, passowrd recovery, bios, etc.
  • Allow tying of information to specific waypoints.
    Each coordinate already exists as a "waypoint" object, with hooks to add photos and notes to these points. Building out supporting features to use these hooks will take time, but will provide a more in-depth experience.
  • Presentation view
    Full screen view with camera flying from waypoint to waypoint, designed for showing a trail to a group or seeing it yourself in a more interactive way.
  • Rework status monitors
    Send notifications to user once certain steps are complete, retrigger failed generation steps based on error information.
  • Dark/light mode
    This is straightforward, but time-consuming and ultimately cut for other features.
